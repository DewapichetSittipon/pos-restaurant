# Snapshot ชื่อเมนู และ soft-delete Menu

ต่อเนื่องจาก [[0003-price-snapshots]] ถ้าเจ้าของร้านเปลี่ยนชื่อเมนูหรือลบเมนู บิล/ใบเสร็จย้อนหลังจะแสดง
ชื่อปัจจุบันผิด หรือทำ FK ของ OrderItem พัง

**ตัดสินใจ:**
- เก็บ snapshot `item_name` ลง OrderItem ตอนสั่ง (คู่กับ unit_price) → ใบเสร็จย้อนหลังถูกเสมอแม้ Menu เปลี่ยนชื่อ
- ห้าม hard-delete Menu ที่เคยถูกสั่ง ใช้ soft delete ผ่าน flag `is_archived` (ซ่อนจากเมนูที่ขาย แต่ยังอ้างอิงได้)

**เหตุผล:** รักษาความถูกต้องของบิลย้อนหลังและ EOD report ที่อ่านประวัติ และกัน FK พัง
