# การ Void OrderItem และการคืนสต็อก

สืบเนื่องจาก [[0001-decrement-stock-at-order-time]] ที่หักสต็อกตอนสั่ง จึงต้องมีทางคืนสต็อก

**ตัดสินใจ:** เฉพาะพนักงาน (admin/kitchen) เท่านั้นที่ void OrderItem ได้ และทำได้เฉพาะขณะสถานะยังเป็น
`pending` เท่านั้น เมื่อขึ้นเป็น `cooking` แล้ว void ไม่ได้ (ของกำลังถูกทำแล้ว) ลูกค้ายกเลิกเองไม่ได้
การ void จะคืน `stock_count` กลับให้ Menu ภายใน transaction เดียวกัน

**เหตุผล:** กันลูกค้ากดยกเลิกพร่ำเพรื่อ/ยกเลิกของที่ครัวเริ่มทำไปแล้ว และรักษาความถูกต้องของสต็อกที่จองไว้ตอนสั่ง
