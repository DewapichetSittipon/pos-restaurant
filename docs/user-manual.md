# คู่มือการใช้งาน (User Manual)

คู่มือการใช้งานระบบ POS ร้านอาหาร + QR Self-Ordering แยกตามบทบาท (role)
ผู้ใช้

> เนื้อหาในแอปแบบย่อดูได้ที่หน้า **"คู่มือ"** บนแถบเมนู (ฝั่งพนักงาน) และปุ่ม
> **"?"** มุมขวาบนหน้าสั่งอาหาร (ฝั่งลูกค้า) เอกสารนี้คือฉบับเต็ม

## ระบบมีผู้ใช้ 5 บทบาท

| บทบาท | เข้าใช้ที่ | สรุปสั้น |
|-------|-----------|----------|
| 🏢 ผู้ดูแลแพลตฟอร์ม | `/platform/login` | ดูแลทุกร้าน สร้าง/อนุมัติร้าน |
| 👑 เจ้าของร้าน (OWNER) | `/login` | ทำได้ทุกอย่างในร้านตัวเอง |
| 🧑‍🍳 พนักงานเสิร์ฟ (WAITER) | `/login` | หน้าโต๊ะ/เช็คบิล/กะ/จองโต๊ะ |
| 🍳 ครัว (KITCHEN) | `/login` | จอครัวอย่างเดียว |
| 📱 ลูกค้า | สแกน QR ที่โต๊ะ | สั่งอาหารเองจากมือถือ |

---

## 🏢 ผู้ดูแลแพลตฟอร์ม (Platform Admin)

ล็อกอินแยกที่ `/platform/login` อยู่เหนือทุกร้าน

- **สร้างร้านใหม่** — กรอกข้อมูลร้าน + บัญชีเจ้าของร้านเริ่มต้น
- **ดูรายการร้านทั้งหมด** ในแพลตฟอร์ม
- **อนุมัติร้านที่สมัครเข้ามาเอง** — ร้านที่สมัครผ่าน `/signup` จะอยู่สถานะ
  `pending` ต้องกดอนุมัติให้เป็น `active` ก่อนถึงใช้งานได้
  (มี badge แจ้งเตือนจำนวนร้านที่รออนุมัติ)
- **รีเซ็ตรหัสผ่านพนักงาน** ของแต่ละร้าน เมื่อร้านลืมรหัส

---

## 👑 เจ้าของร้าน (OWNER)

เห็นและทำได้ทุกอย่างในร้านตัวเอง แถบเมนูครบทุกหัวข้อ

### ผังโต๊ะ / เช็คบิล (เมนู "ผังโต๊ะ")

- **เปิดโต๊ะ** → ระบบสร้าง QR Code อัตโนมัติให้ลูกค้าสแกนสั่งอาหาร
- **คีย์ออเดอร์ให้โต๊ะ** — สั่งแทนลูกค้าได้ (เช่นลูกค้าสั่งที่เคาน์เตอร์)
- **ดูรายการที่โต๊ะสั่ง** และรับเรื่อง "เรียกพนักงาน / เรียกบิล"
- **เช็คบิล** — ส่วนลด · วิธีชำระ (เงินสด/โอน) · เงินทอน · VAT ·
  เซอร์วิสชาร์จ · แลก/สะสมแต้มสมาชิก · QR PromptPay
- **ย้ายโต๊ะ · รวมบิล · แยกบิล** (ลูกค้าจ่ายแยกคนได้)

### ครัว (เมนู "ครัว")

ดูคิวออเดอร์ · เปลี่ยนสถานะอาหาร (รอคิว → กำลังทำ → เสิร์ฟแล้ว) ·
ยกเลิก (void) พร้อมเหตุผล

### กลับบ้าน (เมนู "กลับบ้าน")

สร้างออเดอร์ takeaway ไม่ผูกโต๊ะ · เช็คบิลเหมือนหน้าโต๊ะ

### กะ / เงินลิ้นชัก (เมนู "กะ/ลิ้นชัก")

- **เปิดกะ** — ใส่เงินตั้งต้นในลิ้นชัก
- **ปิดกะ** — นับเงินจริง ระบบคำนวณเงินเกิน/ขาดให้
- ดู **ประวัติกะ** ย้อนหลัง

### จองโต๊ะ (เมนู "จองโต๊ะ")

เพิ่ม/ดูการจองรายวัน · สถานะ จองไว้ → มาแล้ว / ยกเลิก

### รายงานยอดขาย (เมนู "ยอดขาย")

- **สรุปยอดวัน (EOD)** — แยกเงินสด/โอน/VAT/เซอร์วิส
- เมนูขายดี · เวลาเตรียมอาหารเฉลี่ย · ยอดรายชั่วโมง ·
  ยอดช่วงวันที่ (สัปดาห์/เดือน)
- **ดาวน์โหลด CSV** · ดูรายละเอียดบิลย้อนหลัง · พิมพ์ใบเสร็จซ้ำ ·
  **คืนเงิน (refund)**

### จัดการร้าน (เมนู "จัดการร้าน" — มีแท็บย่อย)

- **โต๊ะ** — เพิ่ม/แก้/ลบโต๊ะ
- **หมวดหมู่** — จัดกลุ่มเมนู
- **เมนู** — รูป/ราคา/สต็อก/เปิด-ปิดขาย (`is_available`)
- **ชุด/คอมโบ** — ราคาคงที่รวมหลายเมนู (ครัวเห็นส่วนประกอบแยกชิ้น)
- **ข้อมูลร้าน** — หัวใบเสร็จ · PromptPay · VAT · เซอร์วิสชาร์จ ·
  อัตราแต้มสมาชิก
- **พิมพ์** — ตั้งค่าเครื่องพิมพ์ใบเสร็จ (thermal / ESC-POS)
- **พนักงาน** — เพิ่ม/ลบ/รีเซ็ตรหัส/กำหนด role (OWNER/WAITER/KITCHEN)
- **สมาชิก** — จัดการสมาชิก/แต้มสะสม
- **โปรโมชัน** — กฎส่วนลด (happy hour / BOGO / สมาชิก / วันเกิด)
- **บันทึก (audit log)** — ดูประวัติการกระทำของพนักงาน
- **บัญชี** — เปลี่ยนรหัสผ่านของตัวเอง

## 🧑‍🍳 พนักงานเสิร์ฟ (WAITER)

เห็นเมนู: **ผังโต๊ะ · กะ/ลิ้นชัก · จองโต๊ะ · กลับบ้าน** (และ "คู่มือ")

- ทำงานหน้าโต๊ะได้ทั้งหมด — เปิดโต๊ะ / เช็คบิล / ย้าย / รวม / แยกบิล /
  คีย์ออเดอร์ / void
- เปิด-ปิดกะ · จองโต๊ะ · สร้างออเดอร์กลับบ้าน
- ค้นหา/สมัครสมาชิกตอนเช็คบิล
- ❌ **ไม่เห็น** รายงานยอดขาย และหน้าจัดการร้าน (เมนู/พนักงาน/audit)

---

## 🍳 ครัว (KITCHEN)

เห็นเมนู: **ครัว** อย่างเดียว (และ "คู่มือ")

- ดูคิวออเดอร์ใหม่แบบ **real-time** (เด้งทันทีเมื่อมีออเดอร์เข้า)
- เปลี่ยนสถานะอาหาร: **รอคิว → กำลังทำ → เสิร์ฟแล้ว**
- ออเดอร์ที่กดส่งพร้อมกัน (รอบเดียวกัน) จะรวมเป็น "ticket" เดียว
- **ยกเลิก (void)** รายการพร้อมเหตุผล — คืนสต็อกให้อัตโนมัติถ้ายังไม่เริ่มทำ

---

## 📱 ลูกค้า (สแกน QR — ไม่มีบัญชี)

1. **สแกน QR** ที่โต๊ะด้วยกล้องมือถือ → เปิดหน้าสั่งอาหาร
2. **เลือกภาษา** ได้ (ไทย / English / 中文) มุมขวาบน
3. **เลือกเมนู** → ใส่ตัวเลือก/หมายเหตุ (เช่น ไม่เผ็ด/พิเศษ) → เพิ่มลงตะกร้า
4. กด **ยืนยันสั่ง** — ออเดอร์ส่งเข้าครัวทันที
5. ดู **สถานะอาหารแบบสด** (รอคิว / กำลังทำ / เสิร์ฟแล้ว) ที่ปุ่ม "รายการ"
6. กด **เรียกพนักงาน** หรือ **เรียกเช็คบิล** เมื่อต้องการ

> เปิดดูคำแนะนำได้ทุกเมื่อที่ปุ่ม **"?"** มุมขวาบน

## คำศัพท์ที่ควรรู้ (สรุป)

- **บิล (Bill)** — ยอดรวมต่อโต๊ะ/ออเดอร์หนึ่งครั้ง ตั้งแต่เปิดถึงปิดโต๊ะ
- **รอบ (batch)** — กลุ่มรายการที่กดส่งพร้อมกัน (รอบที่ 1, 2, ...)
- **สถานะอาหาร** — รอคิว (queued) → กำลังทำ (cooking) → เสิร์ฟแล้ว (served)
  + ยกเลิก (voided)
- **เปิด-ปิดขาย (`is_available`)** — สวิตช์ปิดขายชั่วคราว แยกจากจำนวนสต็อก
- ดูคำศัพท์ domain เต็มที่ [CONTEXT.md](../CONTEXT.md)
