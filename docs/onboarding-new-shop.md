# คู่มือเปิดร้านใหม่ (Onboarding)

ขั้นตอนพาร้านอาหารร้านหนึ่งขึ้นใช้งานระบบ สำหรับผู้ดูแลแพลตฟอร์มและเจ้าของร้าน

## ภาพรวมบทบาท

| บทบาท | login ที่ | ทำอะไรได้ |
|-------|----------|-----------|
| Platform Admin | `/platform/login` | อนุมัติร้าน, รีเซ็ตรหัสผ่านพนักงาน, สร้างร้าน |
| OWNER (เจ้าของร้าน) | `/login` | ทุกอย่างของร้าน: ตั้งค่า, พนักงาน, เมนู, ยอดขาย |
| WAITER (เสิร์ฟ) | `/login` | ผังโต๊ะ เปิดโต๊ะ รับออเดอร์ เช็คบิล |
| KITCHEN (ครัว) | `/login` | หน้าครัว คิวอาหาร อัปเดตสถานะ |

## ขั้นตอน

### 1. ร้านสมัคร (2 ทาง)
- **ร้านสมัครเอง:** เปิด `/signup` กรอกชื่อร้าน/ผู้ติดต่อ/เบอร์ + ตั้ง username/password
  → ร้านสถานะ `pending` (login ได้ เห็นหน้า "รออนุมัติ") + ระบบแจ้งเตือนผู้ดูแล (ถ้าตั้ง LINE/Email)
- **ผู้ดูแลสร้างให้:** ที่ `/platform` → สร้างร้าน (ตั้ง `active` ทันที)

### 2. ผู้ดูแลอนุมัติ
ที่ `/platform` กดอนุมัติร้าน `pending` → เป็น `active` ร้านเริ่มใช้งานได้

### 3. เจ้าของร้านตั้งค่าร้าน (`/admin/manage`)
ทำตามลำดับนี้:
1. **ข้อมูลร้าน** — ชื่อ/ที่อยู่/เบอร์ (ขึ้นหัวใบเสร็จ), **เลขผู้เสียภาษี** (ถ้าใส่ ใบเสร็จจะออกเป็น "ใบกำกับภาษีอย่างย่อ"), **PromptPay ID** (รับโอน QR)
2. **VAT / เซอร์วิสชาร์จ** — อัตรา + โหมด VAT (รวมในราคา/บวกเพิ่ม)
3. **แต้มสะสม** (ถ้าต้องการ) — แต้มต่อ 100 บาท
4. **หมวดเมนู** แล้วค่อย **เมนู** (ราคา/รูป/สต็อก)
5. **โต๊ะ** — เพิ่มหมายเลขโต๊ะ
6. **พนักงาน** — เพิ่ม WAITER/KITCHEN ตามต้องการ

### 4. เริ่มขาย
1. `/admin/shift` → **เปิดกะ** (ใส่เงินทอนตั้งต้น)
2. พิมพ์ QR โต๊ะ (จากผังโต๊ะ) วางบนโต๊ะ ลูกค้าสแกนสั่งเอง หรือพนักงานรับออเดอร์
3. สิ้นวัน: **ปิดกะ** (นับเงินจริง กระทบยอด) + ดูรายงานที่ `/admin/report`

## ลืมรหัสผ่าน
- พนักงานที่ยัง login ได้: เปลี่ยนเองที่ `/admin/manage` → บัญชี
- ลืมจน login ไม่ได้: ให้ Platform Admin รีเซ็ตให้ที่ `/platform`

## เช็กลิสต์ก่อนส่งมอบร้าน
- [ ] ตั้งข้อมูลร้าน + เลขภาษี (ถ้ามี) ครบ
- [ ] VAT/เซอร์วิสตรงกับที่ร้านคิดจริง
- [ ] เมนู + ราคาครบ, โต๊ะครบ
- [ ] ทดลองเปิดโต๊ะ → สั่ง → เช็คบิล → ปริ้นต์ใบเสร็จ 1 รอบ
- [ ] เปิด/ปิดกะ ทดสอบกระทบยอด
