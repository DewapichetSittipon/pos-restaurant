# การพิมพ์ใบเสร็จ (Thermal Printing)

## สถานะปัจจุบัน
ใบเสร็จพิมพ์ผ่าน **`window.print()`** (`web/src/utils/printReceipt.ts`) จัดหน้าเป็นความกว้าง 80mm
- ✅ ใช้ได้กับเครื่องพิมพ์ทุกชนิดที่ติดตั้งเป็น printer ของ OS (รวม thermal ที่มีไดรเวอร์)
- ✅ มีเลขที่ใบเสร็จรันต่อเนื่อง + สลับหัวเป็น "ใบกำกับภาษีอย่างย่อ" อัตโนมัติเมื่อร้านใส่เลขผู้เสียภาษี
- ⚠️ บน iPad/มือถือ ต้องผ่าน dialog ปริ้นต์ของ OS (เลือกเครื่องเอง) — ไม่ราบรื่นเท่า direct print

## ทางเลือกพิมพ์ตรงเข้าเครื่อง thermal (ESC/POS) — งานในอนาคต

ถ้าต้องการพิมพ์ทันทีไม่ผ่าน dialog ให้เลือกหนึ่งใน:

1. **WebUSB / Web Bluetooth + ESC/POS** (เบราว์เซอร์ → เครื่องพิมพ์โดยตรง)
   - ส่ง raw ESC/POS bytes ไปเครื่องพิมพ์ USB/Bluetooth
   - ข้อดี: ไม่ต้องมี print server; ข้อเสีย: รองรับเฉพาะ Chrome/Edge, ต้องจับคู่อุปกรณ์, ไม่รองรับ iOS
   - ไลบรารีอ้างอิง: `esc-pos-encoder`

2. **Local print agent** (โปรแกรมเล็กรันที่เครื่องแคชเชียร์)
   - เว็บ POST ใบเสร็จไป `http://localhost:<port>` แล้ว agent ส่ง ESC/POS เข้าเครื่องพิมพ์
   - ข้อดี: เสถียร รองรับทุกเบราว์เซอร์/รุ่นเครื่องพิมพ์; ข้อเสีย: ต้องติดตั้ง agent ต่อเครื่อง

3. **Cloud print (เครื่องพิมพ์รองรับเอง)** เช่น Star CloudPRNT — เครื่องพิมพ์ดึงงานจาก endpoint เราเอง
   - ข้อดี: ไม่ต้องลงอะไรที่หน้าร้าน; ข้อเสีย: ต้องใช้เครื่องพิมพ์รุ่นที่รองรับ

## ข้อแนะนำ
เริ่มจาก `window.print()` (ที่มีอยู่) ให้ผ่านงานจริงก่อน แล้วค่อยเพิ่ม **ทางเลือกที่ 2 (local agent)** เมื่อร้านต้องการความเร็ว — เป็นทางที่ทนทานและรองรับฮาร์ดแวร์กว้างสุด

จุดต่อยอดในโค้ด: สร้างฟังก์ชันคู่ขนานกับ `printReceipt()` ที่ encode ESC/POS แทนการ render HTML โดยใช้ข้อมูลชุดเดียวกัน (`CheckoutResult`)
